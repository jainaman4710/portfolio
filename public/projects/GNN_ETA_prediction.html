The Core Problem

OSRM treats every trip as independent. It has no concept of whether a hub is a structural chokepoint, whether a corridor has historically run 3× over estimate, or whether Friday evenings are consistently worse than Tuesday mornings.

The result: systematic ETA underestimation, missed SLAs, and capacity planning failures that cascade across the entire network.

The Graph Approach

We model the logistics network as a directed graph G = (V, E) where nodes are facilities and edges are corridors. A Graph Attention Network learns which neighbours matter most for predicting delay at any given hub.

This spatial awareness: knowing where a hub sits in the network, not just its own history: is the primary source of predictive gain over tabular baselines.

Key EDA Findings

⚠️ Severe Right Skew in Factor

Most trips cluster between 1.2–2.5×, but outliers extend beyond 30×. Extreme cases reflect facility breakdowns, route closures, or OSRM miscalibration: not noise. Huber loss was chosen specifically for robustness against these outliers.

🕐 Real Temporal Signal

Night departures show consistently higher delay. A simple integer hour encoding would fail (23 and 0 appear maximally different when they're adjacent). This motivated the cyclic sine/cosine encoding used in the final model.

🗺️ Geographic Concentration Risk

Maharashtra, Karnataka, and Gujarat account for a disproportionate share of both volume and delay incidents. This concentration means hub-level structural analysis: not just trip-level features: is essential for meaningful SLA management.

What is Betweenness Centrality?

Betweenness centrality measures how often a hub lies on the shortest path between other hubs in the network. A hub with high betweenness is a structural junction: many routes pass through it. If it slows down, many downstream deliveries slow down too.

The composite risk score multiplies this structural importance by the hub's actual historical delay factor. A hub that is both central AND slow is the highest-priority target for intervention.

Why Cyclic Encoding?

A plain integer hour (0–23) creates a false discontinuity: hour 23 and hour 0 appear maximally different, yet they are adjacent in time. Cyclic encoding solves this:

hour_sin = sin(2π · hour / 24) day_sin = sin(2π · day / 7) day_cos = cos(2π · day / 7)

This ensures 11PM and midnight are mathematically close: critical for capturing overnight departure delay patterns correctly.

Backward Feature Elimination: Test MAE by Feature Count
BFE iteratively removes the least useful feature. Optimal at 8 features. Adding start_date, is_night, start_hour_cos increases test MAE.
Model Architecture
From Baselines to Graph Attention Networks
Three GNN architectures developed iteratively, each addressing a specific limitation of the previous. Three tabular baselines provide the benchmark floor.
BASELINES

Linear Regression · Random Forest · XGBoost

Each trip treated as an independent row. No network topology. Best baseline (XGBoost) achieves MAE 1.133, MAPE 41.78%, Hit Rate@15% of 29.63%. These establish the floor the graph models must beat: and beat convincingly.

ITERATION 1

GCN Baseline: Graph Convolutional Network

First graph model. Introduces spatial awareness by aggregating neighbour features. Key limitation: GCN uses fixed, normalised adjacency weights: it cannot distinguish which neighbours matter more. Every neighbouring hub is treated equally regardless of how much traffic it contributes or how often it delays.

ITERATION 2

ST-GAT with Residual Connections: GATv2Conv

Replaces GCN with GATv2Conv, which learns attention coefficients α_ij: the model discovers which neighbouring hubs carry more predictive signal for a given prediction. A residual skip connection (h_out = GAT(h) + Linear(h_in)) preserves raw spatial features through deep aggregation, preventing gradient loss and retaining local hub characteristics.

DEPLOYED: BEST MODEL

ST-GAT Iteration 3: Full Architecture with Regularisation

Two GATv2 layers (4 heads → 1 head), residual skip, MLP prediction head. Dropout 0.3 + L2 weight decay 1e-4. Trained 200 epochs with Huber loss for outlier robustness. Adam optimiser, learning rate 1e-3. Achieves MAE 0.713, MAPE 21.11%, Hit Rate@15% of 52.57%: nearly double XGBoost.

ALTERNATIVE ARCHITECTURE

Decoupled GAT-LSTM: Parallel Spatial + Temporal Modelling

Separates spatial modelling (GATv2) from temporal modelling (LSTM). Despite explicit temporal sequence learning, underperforms ST-GAT Iteration 3 (MAE 0.789 vs 0.713). Conclusion: for this dataset, cyclic feature encoding captures temporal signal better than an LSTM: the sequence is too short to benefit from recurrence.

GATv2 Attention Mechanism

e_ij = a^T · LeakyReLU(W · [h_i ‖ h_j ‖ edge_ij]) α_ij = softmax_j(e_ij) h_out = σ(Σ_j α_ij · W·h_j) + Linear(h_in)

α_ij tells us how much hub j's state should influence hub i's prediction. A hub that frequently delays its neighbours gets higher attention weights: the model learns network delay propagation patterns automatically.

Strategic Recommendations
From Model Output to Operational Decisions
The analysis translates directly into five operational decisions for Delhivery's Head of Network Operations: each grounded in graph metrics and historical data, with a clear rationale and trade-off.
1

Priority Hub Upgrades: IND712311AAA & IND562132AAA

These two hubs combine the highest betweenness centrality with the highest delay factors (3.0× and 2.0× respectively). Capacity expansion and faster dwell-time processing at these facilities reduces SLA breach propagation across the broadest set of downstream corridors. Trade-off: significant capex, but highest network-wide impact per investment.

2

Shift Chronic Breach Corridors to FTL

Corridors with median factor above 2.0× and more than 10 historical trips are candidates for mandatory FTL routing. Eliminating intermediate hub stops removes dwell-time accumulation: the primary driver of delay amplification on multi-hop Carting routes. Trade-off: higher per-trip cost, offset by SLA compliance improvement.

3

Recalibrate OSRM Baselines for Outlier Corridors

A subset of corridors show 500%+ excess over OSRM: this reflects baseline miscalibration rather than pure operational delay. Correcting these baselines improves SLA tracking accuracy and eliminates false breach alerts that inflate operational response costs unnecessarily.

4

Deploy ST-GAT in Production ETA Pipeline

The model achieves 21.1% MAPE vs 41.8% for XGBoost. CPU-only inference (no GPU required). Replacing OSRM-only estimates with ST-GAT predictions for the top 500 corridors by volume materially reduces customer-facing ETA errors with manageable infrastructure cost.

5

Implement Time-of-Day Routing Rules

Hour-of-day sensitivity analysis shows 10–15% ETA improvement by scheduling FTL departures during low-congestion early morning windows (6AM–10AM). This is a zero-capex intervention: pure scheduling discipline recovers significant delay on high-factor corridors.

Limitations & Future Work

Current Limitations

  • Single-day dataset (Sept 2018): limited longitudinal seasonal variation
  • No real-time traffic integration: model uses historical medians
  • Scaled features in training require consistent preprocessing in production
  • ST-GAT with BFE was marginally edged by ST-GAT Iter. 3 on some metrics with the current dataset size. With substantially more data, BFE's leaner feature set is likely to generalise better and outperform the full feature set more decisively.

High-Value Next Steps

  • GraphSAGE / node2vec for inductive generalisation to new hubs
  • Multi-day rolling training to capture week-of-year seasonality
  • Binary SLA breach prediction as a downstream classification task
  • Real-time graph anomaly detection for proactive rerouting
