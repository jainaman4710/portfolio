Product Requirements Document · Grammarly · B2C Premium

The missing layer
in AI writing.

PromptAssist is an opt-in, on-demand feature for the Grammarly browser extension and Grammarly Editor. It reads a user's in-progress text, infers intent and context, and generates a structured, high-quality prompt — injected directly into any AI tool with explicit user approval.

01 Vision + Mission

Why this matters now.

As AI writing tools become mainstream, effective communication now includes communicating effectively with AI. Grammarly has historically focused on the quality of what a user writes. PromptAssist extends that mission into a new domain.

"The differentiator is no longer having access to AI. It's being able to use it well."
The behaviour change parallel
Grammar checker (established)
Automated grammar suggestions
Consistent exposure to suggestions gradually reduced users' felt need to consciously review their own grammar. The habit became automatic.
PromptAssist (new frontier)
Automated prompt engineering
The same ambition applied to prompting. Write your thoughts naturally; Grammarly translates intent into high-quality instructions for any AI tool.
Strategic framing
Mission alignment

Grammarly's mission to improve lives by improving communication now extends to improving how users communicate with AI tools. PromptAssist is the natural evolution of that mission into the AI-assisted writing era.

North star UX

The user writes their thoughts naturally, without worrying about crafting an AI prompt. Grammarly translates that intent into high-quality instructions for any AI tool. Writing without the cognitive overhead of prompt engineering.

03 Problem Statement

The friction loop.

The quality of AI output is directly proportional to the quality of the prompt — a skill most users have not developed and do not want to learn.

The re-prompting cycle most users are stuck in
✍️
Write vague prompt
User types a rough, underspecified instruction
›
🤖
Generic AI output
AI responds technically but misses what was actually needed
›
😤
Frustration
User blames the AI, not the prompt
›
🔄
Guess and retry
User re-prompts without understanding what to change
Repeat until frustration overrides productivity
Three specific pain points
🎯
Ineffective technique
Users rely on vague instructions, over-specify irrelevant details, or prompt as if using a search engine rather than a generative model.
⚡
Iteration fatigue
Users who don't know why a prompt failed can't improve it meaningfully. Each re-prompt is essentially a guess, compounding frustration over sessions.
🔗
No continuity
Users repeat context unnecessarily, fail to build on prior AI responses, and treat each prompt in isolation from the conversation so far.
Why this matters now

If users spend more time wrestling with AI than writing, Grammarly risks becoming irrelevant to the most important writing workflow shift in a generation. PromptAssist positions Grammarly as the intelligent orchestration layer for the full AI writing workflow, not just the final proofreading step.

04 Solution Overview

What PromptAssist does.

An opt-in feature that activates within the user's existing AI tool workflow, not before it. The user begins working in an AI tool; when they want a better-structured prompt, they invoke PromptAssist.

6

User reviews and sends. PromptAssist's role is complete.

05 Metrics + KPIs

How we know it's working.

Metrics are structured in three layers. Adoption is measured first, then quality depth, then downstream business outcomes. Each layer unlocks at a different rollout phase.

★
North Star Metric
Prompt Acceptance Rate
The percentage of PromptAssist-generated prompts approved and injected by the user (versus dismissed), among users who have used the feature at least once in the measurement period. A high rate means users trust the generated prompt enough to send it to an AI tool. A low rate signals poor quality, unclear UX, or accidental triggering.
L1 — Adoption
Tracked from Phase 2 · Closed Beta
Feature adoption rate
% of active Premium B2C users who use PromptAssist at least once within 14 days of availability
Monthly active users
Count of unique users who use PromptAssist at least once per calendar month
L2 — Quality + depth
Tracked from Phase 3 · Open Beta
Session depth
Avg. Grammarly interactions per session: PromptAssist users vs matched non-users
Modifier usage rate
% of accepted prompts where the user applied at least one modifier chip
Feature CSAT
In-app survey score 7 days after first PromptAssist use, specific to the feature
Prompt edit rate
% of generated prompts the user edits inline before approving
L3 — Business outcomes
Tracked 3 months post Full GA
Premium retention differential
Churn rate: active PromptAssist users vs matched Premium non-users. Needs sufficient cohort — measured 3 months post-GA.
Premium conversion lift
Conversion rate Free to Premium for users shown the PromptAssist capability vs those who were not

V1 scope decision: platform optimisation deferred

Different LLMs respond differently to the same prompt. For V1, PromptAssist generates a single LLM-agnostic prompt following universal best practices. Platform-specific optimisation — generating subtly different prompts for Claude vs ChatGPT — is a post-V1 enhancement. This decision significantly reduces V1 AI scope while still delivering core user value.

Phase 1
Internal Dogfooding
Weeks 1 to 4 · All Grammarly employees
Daily real-work tasks using PromptAssist once per day (not test files). Structured scenarios across email, project update, and essay formats. Prompt quality rated on a 1 to 5 scale: intent match and willingness to submit without editing. End-of-phase 10-question survey covering clarity, trust, perceived value, and willingness to recommend.
Exit criteria
Avg. prompt quality ≥ 3.8/5 · Injection success ≥ 95% on Chrome for ChatGPT + Gemini · No P0 or P1 bugs open at end of Week 4
Phase 2
Closed Beta
Weeks 5 to 10 · Random 5% of active Premium B2C (stratified: student, professional, general writer)
Feature enabled for selected users with a one-time in-app tooltip on first login. Weekly monitoring of Prompt Acceptance Rate, Feature Adoption Rate, and 7-day post-first-use CSAT. Qualitative interviews with 10 to 15 beta users in Week 4, focused on why prompts were edited or dismissed.
Exit criteria — Go / No-Go review end of Week 10
Prompt Acceptance Rate ≥ 50% · CSAT ≥ 3.8/5 · No P0 or P1 issues outstanding
Phase 3
Open Beta
Weeks 11 to 16 · 30% of Premium B2C
Discovery pop-up activated for the first time on ChatGPT.com, gemini.google.com, and claude.ai. Infrastructure scaling validated — engineering monitors p50, p90, and p99 latency and error rates under increased load. Model retraining initiated if Prompt Acceptance Rate falls below 55% in any week.
Phase 4
General Availability
Week 17 onwards · All active Premium B2C
Pop-up activated for all Premium users visiting supported AI tool sites. Free-tier users see the toolbar button as disabled with an upgrade prompt. L3 retention differential measurement begins 3 months post-GA once the active user cohort is large enough to be statistically meaningful.

09 Privacy + Data Handling

Privacy as a design constraint.

Privacy is foundational, not an afterthought. The following commitments govern data handling and must be reflected in both engineering implementation and every user-facing communication.

→
What PromptAssist reads
  • →Current input text — only at the moment the user triggers PromptAssist. No passive monitoring of the input field at any time.
  • →Session conversation history — only with explicit opt-in consent. Retained in browser memory only; never transmitted to Grammarly servers.
×
What PromptAssist never does
  • ×Stores raw user text server-side. Processed within the session and discarded.
  • ×Logs or stores generated prompts unless the user explicitly saves one.
  • ×Has any visibility into AI responses or user interactions within third-party tools.
  • ×Writes session conversation history to disk or transmits it to any server.
User controls
🔒
Off by default
Users must actively enable via first-use modal or Grammarly Settings.
⏹️
Disable anytime
Disabling in Settings stops all input reading associated with this feature immediately.
🧹
Session awareness toggle
Separate opt-in. Turning off clears any session history held in memory immediately.
✋
Withdraw feedback consent
Available at any time via Settings › Privacy › PromptAssist.
10 Risk Register

What could go wrong.

Technical and product risks documented with likelihood, impact, and mitigation strategy. Each mitigation is actionable, not aspirational.

Technical risks

11 Open Questions

Unresolved before development.

Three questions requiring a decision before engineering kickoff. Each has direct implications for scope, architecture, or go-to-market.

OQ 01
Free vs Premium gating
Should PromptAssist be exclusively Premium, or should a limited version (3 uses per day) be available on the Free tier as a conversion driver? This decision impacts engineering (usage metering infrastructure) and GTM (free-tier discovery placement and messaging).
OQ 02
Minimum viable text length
If the user has written fewer than approximately 20 words, the context model will have insufficient signal. Should the feature be disabled below a word threshold, or should it present a generic template prompt in this case? The answer affects both the AI pipeline and the overlay UX.
OQ 03
Injection restriction contingency
What is the strategy if a major AI provider explicitly restricts browser extension injection into their input fields? Clipboard fallback mitigates user impact but significantly affects the core UX and perceived value of the feature. A formal contingency plan is needed before GA.
12 Future Scope — Post V1

Where it goes next.

Five post-V1 enhancements identified. None are in V1 scope but are documented here to inform architecture decisions and avoid building in directions that foreclose them.

🎯
Platform-specific optimisation
Generate subtly different prompts tailored to ChatGPT, Claude, and Gemini based on known differences in how each model responds to instruction style.
🏢
B2B expansion
Team admins define company-voice prompt templates. PromptAssist applies them automatically by document type, transforming a B2C feature into an enterprise content governance tool.
🎛️
Goal-based mode
User selects an explicit goal before generation: Improve this paragraph · Generate new content · Change the tone. More precise targeting without relying entirely on text inference.
📱
Mobile keyboard
Extend PromptAssist to the Grammarly iOS and Android keyboard for prompt generation in mobile writing contexts.
🔌
Native API integration
As LLM providers open richer APIs, move from browser UI injection to direct API-level calls, enabling full control over model parameters and output format.
