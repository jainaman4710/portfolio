Key Challenges

Class Imbalance

5,536 normal vs. 21 fraud transactions in the sample. Standard accuracy becomes misleading; a model predicting "always normal" scores 99.6% accuracy trivially.

Dynamic Fraud Patterns

Fraudulent behaviour mimics legitimate transactions, making signal separation difficult without careful feature engineering.

Natural Outliers

High transaction amounts are not measurement errors; they are genuine signals. Outlier removal would destroy fraud indicators.

03

Feature Engineering

Since raw correlations were too weak for model building, 8 feature groups were engineered from the raw variables. Post-engineering, correlations with is_fraud improved substantially.

① Time & Date Decomposition

Split trans_date_trans_time into week number, month, year, hour, minute, and second. Enables the model to detect temporal fraud patterns (e.g., transactions at unusual hours).

② Hour Encoding (Abnormal Flag)

Binary feature: hourEnc = 1 if transaction occurs between 21:00 to 05:00 (abnormal hours). Captures the increased fraud risk during off-peak hours.

③ Transaction Frequency Windows

Rolling count of transactions in the last 1, 7, and 30 days, aggregated by both cc_num (cardholder) and merchant. Unusual burst activity is a classic fraud signal.

④ Time Since Last Transaction (Recency)

Time delta (in seconds) since the same cardholder's previous transaction. Very short gaps suggest rapid automated fraud. NaN values (first transactions) filled with 0.

⑤ Age at Transaction Time

Computed as the difference between transaction datetime and date of birth, giving age at the time of the transaction, not current age. This avoids data leakage.

⑥ Gender One-Hot Encoding

Binary encoding: Male = 1, Female = 0. This converts the categorical gender variable into a numerical format suitable for all ML models.

⑦ Haversine Distance

Geographic distance (miles) between cardholder's home location and merchant using the Haversine formula. Large distances between cardholder and merchant are a strong fraud indicator.

⑧ Category One-Hot Encoding

14 merchant categories expanded into binary dummy variables. Nominal data cannot be passed ordinally to ML models; one-hot encoding prevents false ordinal relationships.

04

Feature Scaling

Features like amt have large ranges that could dominate distance-based models. Three scaling methods were compared before selecting the most appropriate one.

MinMax Scaler
Compresses all values into [0, 1] range by subtracting the minimum and dividing by range.
✓ Bounded output, intuitive interpretation
✗ Highly sensitive to outliers; one large fraud amount distorts the entire scale
Standard Scaler
Subtracts mean and divides by standard deviation, producing z-scores with mean 0, std 1.
✓ Works well when data is approximately Gaussian
✗ Assumes normal distribution; fraud amounts are right-skewed, violating this assumption
Robust Scaler
Uses IQR (interquartile range) instead of mean/std. Subtracts median and divides by IQR.
✓ Robust against outliers; preserves natural fraud signals in high-value transactions
✗ Slightly less interpretable than MinMax
Decision: Robust Scaler was selected because the dataset contains natural outliers (high-value transactions) that must not be distorted. MinMax would have compressed fraud signals; Standard assumed normality the data does not exhibit.

Why SMOTE wins

Generates new synthetic fraud samples by interpolating between existing ones in feature space (not duplicates). Avoids information loss (undersampling) and overfitting (oversampling).

Decision rationale: Despite undersampling having the highest amt-fraud correlation (0.763), it discards 99.6% of data. SMOTE preserves all original data while generating diverse synthetic fraud instances, making it the most sound statistical choice for a real fraud detection system.
06

Model Descriptions

Four classical ML classifiers plus a Deep Neural Network were trained on the SMOTE-balanced dataset with an 80/20 train-test split.

Gaussian Naïve Bayes

Probabilistic classifier based on Bayes' theorem: P(h|d) = P(d|h)·P(h) / P(d). Assumes features are independent and follow a Gaussian distribution. Computationally lightweight but oversimplified for correlated fraud features. Acts as a useful baseline.

Random Forest (max_depth=4)

Ensemble of decision trees trained via bagging on random feature subsets. Each tree votes and the majority class wins. Depth is capped at 4 to prevent overfitting. Highly robust to outliers and non-linear relationships.

K-Nearest Neighbors (k=2)

For each new transaction, finds the 2 most similar transactions in the training set and predicts the majority class. Optimal k=2 was determined empirically. Sensitive to scale, hence Robust Scaling was critical here.

XGBoost Classifier

Gradient boosting framework: sequentially trains shallow trees, each correcting the residuals of the previous. Evaluated with aucpr metric. Typically superior to Random Forest due to boosting vs. bagging, and its ability to learn from its own errors.

Deep Neural Network (5-layer FC)

Sequential Keras model: Input(14) → Dense(14, ReLU) → Dense(24, ReLU) → Dropout(0.5) → Dense(24, ReLU) → Dense(24, ReLU) → Dense(1, Sigmoid). Trained with Adam optimiser, binary cross-entropy loss, 20 epochs, batch size 10. The 0.5 dropout layer prevents overfitting by randomly deactivating neurons during training.

Evaluation Metrics: Why Each Matters for Fraud Detection
TPR (Sensitivity) ↑ Maximise
A missed fraud (False Negative) means a real customer gets defrauded with no detection. This is far worse than a false alarm. TPR = TP/(TP+FN).
FPR ↓ Minimise
False positives (normal transactions flagged as fraud) inconvenience customers and create unnecessary alerts. Low FPR = high specificity. FPR = FP/(FP+TN).
F1 Score ↑ Maximise
Harmonic mean of Precision and Recall; balances the tradeoff between catching all fraud and not over-flagging. F1 = 2·(P·R)/(P+R).

Why XGBoost outperforms the rest

Boosting sequentially corrects model errors; each new tree focuses on the hard-to-classify fraud cases. Unlike bagging (Random Forest), it does not treat all errors equally, giving it superior discrimination power for the rare fraud class.

Role of Feature Engineering

Engineered features (time_diff, hourEnc, transaction frequency windows, Haversine distance) were critical. The correlation jump from 0.17 (raw amt) to 0.763 (post-SMOTE) confirms that a well-engineered feature set can multiply model effectiveness.

SMOTE was the right call

Generating synthetic fraud samples rather than copying or deleting allowed all five models to learn genuine fraud patterns, not memorise a tiny number of real fraud cases.

Final Takeaway  +  XGBoost has been determined to be the best and most suitable ML model for credit card fraud detection in this project, achieving near-perfect scores across every evaluation metric. The combination of Robust Scaling, SMOTE balancing, and 8 engineered feature groups created the conditions for this result. Future improvements could include hyperparameter tuning via grid search, real-time streaming feature computation, and testing on the full 555K transaction dataset.
